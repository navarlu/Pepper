How this flow works:
- Session manager creates one permanent room (`pepper-main`)
- Listener and user client read tokens from `web/agents-playground/token-latest.json`
- User client publishes external microphone audio as `user`
- On user speech activity, session manager explicitly dispatches agent `Pepper`
- Listener reports agent activity while Pepper audio is playing
- After idle timeout, session manager removes the agent and returns to idle
